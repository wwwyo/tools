# 五滴（goteki）

画像を貼り付けると、使用色の面積比 top5 を五滴抽出するツール。抽出した色は背景に「白い紙に滲んだインク」として散る。

## 使い方

- 画像をドラッグ&ドロップ / クリックで選択 / Cmd+V で貼り付け
- 「サンプル画像で試す」でページ内生成のサンプルを流せる
- HEX / RGB / CMYK をトグルで切り替え。各色 copy と一括 copy あり
- 解析はすべてページ内 Canvas で完結し、画像は外部に送信しない

## 色抽出ロジック（colorQuantize.ts）

1. **縮小サンプリング**: 長辺を縮めてから `getImageData`。全画素を舐めない
2. **バケット量子化**: RGB を粗く丸めて頻度カウント。ほぼ透明な画素は除外
3. **代表色**: バケット中心値ではなく、バケット内実ピクセルの RGB 平均
4. **近似色マージ → top5**: 頻度順に走査し、近すぎるクラスタは面積を合算する。量子化の仕切り境界で1色が割れるのを拾い直すための後処理

距離は知覚的でない素朴なユークリッド。厳密にやるなら Lab + ΔE だが、この用途では過剰なので採用していない。

## インク背景（main.ts renderBlobs）

レイヤーは `インク(z-0) → 白ウォッシュ(z-10) → コンテンツ(z-20)` の3層。

- **1滴 = 要素1個**。層を重ねない。同じ色を重ねると重なり領域だけ乗算で濃くなり、その境界が輪として見えてしまう。現実のインクは1つの液体が広がる現象で、層に分かれていない
- **広がりは要素の拡大ではなく濃度分布の前進**。要素サイズは最終サイズで固定し、`@property` で登録したカスタムプロパティで radial-gradient の stop 位置を動かす。要素ごと拡大すると、輪郭も濃度分布もノイズも相似形のまま大きくなり、滲みではなくズームに見える
- **速度は Washburn 則**（`r² = r₀² + Kt`）に沿わせる。中心のコアは早期に飽和させ、以降は濡れ前線だけが進む。一律の easing では機械的に見える
- **個体差は無相関な乱数にしない**。紙の繊維方向は五滴で共有し、その周りで滴ごとに角度・異方性・速度・到着タイミングをばらす。全部を独立に振ると紙の物性まで滴ごとに変わって見える
- **質感**は SVG フィルタ（feTurbulence + feDisplacementMap の2段）。displacement 量は滴の直径に比例させる。px 固定だと小さい滴で形が崩れ、フィルタ領域の余白を超えて縁が矩形に切れる
- **異方性**は transform ではなく要素の縦横比で出す。transform で潰すとフィルタのノイズ＝紙目まで一緒に伸縮してしまう
- **面積配分**は ratio に比例させつつ指数を圧縮する。素直に面積 ∝ ratio だと上位色が支配的になり、下位の滴が視認できないほど潰れてしまう

調整値（塗り面積、alpha、duration など）は main.ts の名前付き定数に意図つきで置いてある。ここには写さない。

## ファイル構成

```
goteki/
├── index.html        エントリ（title / description は一覧自動生成に使われる）
├── main.ts           UI・インク描画・コピー・フォーマット切替
├── colorFormat.ts    HEX / RGB / CMYK 変換
└── colorQuantize.ts  縮小・量子化・マージ・top5 抽出
```
